# Agent Memory 设计思考

在做智能体（Agent）相关学习与实践时，越来越意识到 Memory 模块是 Agent 能否稳定、连贯、可控的关键。
简单记录一下目前对 Agent 记忆体系的理解与设计思路。

## 1. 为什么 Agent 需要 Memory
- 维持多轮对话上下文，避免“失忆”
- 减少重复检索与重复调用 LLM，降低 token 成本
- 让决策、工具调用、规划具备连续性
- 结合 RAG 实现外部长期知识记忆，降低幻觉

## 2. 记忆分层思路
1. **短期记忆（Short-Term Memory）**
   - 保存最近 N 轮对话历史
   - 控制长度，防止上下文溢出
   - 结构简单，存取速度快

2. **中期记忆（Medium-Term Memory）**
   - 对历史对话做摘要、压缩
   - 保留关键意图、关键结论、关键约束
   - 用于长对话场景下的信息精简

3. **长期记忆（Long-Term Memory）**
   - 外部化存储：向量库 / 数据库
   - 存放业务规则、历史案例、知识库内容
   - 通过检索方式“唤醒”相关记忆

## 3. 记忆模块设计原则
- 可控：记忆内容可审计、可追溯
- 可裁剪：超长历史自动截断，避免模型混乱
- 可检索：支持语义+关键词混合召回
- 可扩展：支持后续接入总结、反思、遗忘策略

## 4. 后续要验证的点
- 不同记忆长度对回答准确性的影响
- 摘要记忆 vs 原始历史记忆的效果差异
- 结合 RAG 时，记忆与检索结果的融合策略
